# 端到端实测报告（阶段三收尾）

> 实测日期：2026-07-07 ｜ 项目：`ChemInfoBot` ｜ 仓库已同步至 GitHub

## 一、实测怎么做的

端到端实测分两层，互不依赖你的私密凭据：

| 层 | 方法 | 验证什么 |
|----|------|---------|
| **离线全链路** | 用 mock 替代 DeepSeek/SMTP/PushPlus，跑真实的 `send_enhanced()` 主路径 | 采集→分析→报告→邮件→推送→落库 代码衔接是否正确 |
| **真实数据层** | 直接调东方财富/腾讯接口；用真实返回体做静态解析验证 | 数据源是否真能用、解析是否对真实数据无误、失败是否编造 |

## 二、实测发现的真实问题（这一步最关键）

离线逻辑全链路 **16/16 PASS**，但真实数据层一打就现原形——原项目（`Codex` 写的那版）的数据源**已经失效**：

| # | 现象 | 根因 | 严重度 |
|---|------|------|--------|
| Bug1 | 涨跌分布 / 板块接口 `data.eastmoney.com/api/qt/clist` 返回 **404** | 东方财富已将 `clist` 接口迁移到 `push2.eastmoney.com` 前缀，代码还在用旧前缀 | 高（全市场涨跌/板块拿不到） |
| Bug2 | 指数接口 `push2…/ulist.np` 偶发被服务器断开，且**解析失败时不降级** | 只网络失败才降级腾讯，解析异常直接 `return None` | 高（指数也拿不到） |
| Bug3 | 腾讯兜底虽接口通，但**字段索引取错**（把今开当涨跌幅） | 解析索引假设错误，`chg_pct` 取到 `4019.49` 远超 ±15  sanity-check → 全被过滤成 `None` | 高（腾讯兜底形同虚设） |

> 也就是说：**代码逻辑全通，但真实数据一个都抓不到**——全靠降级也救不回来。这正是你最初最在意的"数据真实性 / 编造"问题：不是编造，而是**静默失败**，上线后每天推一份缺失报告，比编造更隐蔽。

## 四、当前环境的一个限制（重要）

实测过程中因**短时间密集探测**（精细探测 6 个 URL + 多次 probe），本机出口 IP 被东方财富和腾讯**限流**（返回 `RemoteDisconnected`）。这是探测行为触发的风险控制，**不是代码问题**——裸 `requests` 已证明腾讯接口 200、数据格式正确。

**真实每日单次运行（`run_daily.py` 经任务计划触发）不会触发此限流**，修复后正常抓取。若想在本机立即复测真实抓取，建议隔一段时间（或换网络）再跑 `probe_datasrc.py`。

## 五、填 key 后真跑清单

代码已修好，但**真推送**需要你填 3 类凭据到 `config.py`（当前是占位符）：

| 配置项 | 填什么 | 怎么获取 |
|--------|--------|---------|
| `DEEPSEEK_API_KEY` | `sk-...` | platform.deepseek.com → API keys → 创建（充值约 ¥10 够用很久） |
| `EMAIL_CONFIG.sender_email` / `recipient_email` | 你的 163 邮箱 | 如 `xxx@163.com` |
| `EMAIL_CONFIG.sender_password` | **163 SMTP 授权码**（不是邮箱登录密码） | 163 邮箱网页版 → 设置 → POP3/SMTP → 开启 → 获取授权码 |
| `WECHAT_CONFIG.enabled` | `True`（当前是 `False`） | 开启微信推送 |
| `WECHAT_CONFIG.pushplus_token` | PushPlus token | www.pushplus.plus → 微信扫码登录 → 一键复制 token |

填好后真跑（二选一）：

```bash
# 方式 A：经任务计划入口（会判交易日，非交易日跳过）
python run_daily.py

# 方式 B：手动立即跑一次完整链路
python -c "import stock_mail_plugin; print('OK' if stock_mail_plugin.send_enhanced() else 'NO DATA')"
```
