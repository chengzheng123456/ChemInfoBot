# ChemInfoBot 阶段一改造说明

## 背景

原项目 A股爬虫（`a_stock_spider.py`）在数据源（东方财富）不可达时，会返回**写死的假指数/涨跌数据**（`_demo_market()` 等），且报告 footer 署名"数据来源：东方财富"，把假数据冒名真实来源推送到微信/邮箱。这正是"编造数据"隐患——比参考项目 DSA 更严重（DSA 失败返回 None，原项目失败返回 mock）。

阶段一按 **P0→P4** 在原项目 `chengzheng123456/ChemInfoBot` 上修复并增强。原则：自托管 / 零成本 / 本地 / 数据掌控在自己手里。

## 改动清单

### P0 修编造隐患（最高优先级）
- 删除 `_demo_market` / `_demo_breadth` / `_demo_north_flow` / `_demo` / `_demo_news` 全部函数
- 所有 fetch 失败路径改为返回 `None`（行情类）或 `[]`（新闻类），**绝不伪造数字**
- 文件：`a_stock_spider.py`

### P1 溯源落库
- `data_storage.py` 新增 `stock_market_snapshot` 表（含 `source` / `fetched_at` / `data_complete` 溯源字段 + 各模块 JSON）
- 新增 `save_market_snapshot()` 方法
- `fetch_previous_trading_day_analysis()` 聚合时附加 `source` / `fetched_at` / `data_complete` 元数据
- `stock_mail_plugin.send_enhanced()` 推送前落库快照，可复盘溯源

### P2 多源降级
- 指数行情：东方财富为主，失败时降级腾讯 `qt.gtimg.cn`（带字段合理性校验，拒绝垃圾值）
- 新增 `_fetch_tencent_indices()`

### P3 熔断
- 新增 `CircuitBreaker` 类（连续失败阈值 3、冷却 300s）
- 东方财富 / 腾讯各持独立 breaker，避免雪崩

### P4 footer 真实署名
- `market_report.py` footer 改为按 `data_complete` 动态署名：完整 → "数据来源：东方财富"；缺失 → "（部分数据获取失败，以下内容仅供参考）"，不再无条件冒名
- `notification_sender.py` 推送内容顶部加 "⚠️ 部分市场数据获取失败" 提示
