## 一、目标

1. 把原项目"纯规则 HTML 模板"的报告，升级为**基于真实行情数据的 DeepSeek AI 研判**。
2. 叠加**确定性 guardrail 护栏**，防止 LLM 过度承诺 / 在数据缺失时胡说。

4. LLM 研判结论一并落库，便于复盘。

## 三、analyzer.py 设计

### 3.1 上下文装配 `build_context_pack`
只把阶段一采集的**真实字段**转成文本喂给 LLM。任一字段为 `None`/空时显式标注「未获取」，System Prompt 强制 LLM 不得臆测。

### 3.2 DeepSeek 调用 `_call_deepseek`
- 端点：`https://api.deepseek.com/chat/completions`（OpenAI 兼容）
- 主模型：`deepseek-chat`（日常分析，成本约 ¥1/百万输入、¥2/百万输出）
- 可选复核：`DEEPSEEK_VERIFY_MODEL`（如 `deepseek-reasoner`），对关键决策二次推理
- `response_format: json_object` 强制结构化输出

### 3.3 确定性护栏 `apply_guardrail`（不依赖模型强推理）
| 触发条件 | 动作 |
|----------|------|
| 主要指数单日 ≤ -1.5% / 涨跌比恶化（跌>涨×1.5）/ 北向净流出 ≥50亿 | 大盘偏弱 → 把「买入」软化为「观望」 |
| `data_complete=False`（部分源失败） | 禁止「买入」（降为「持有」），置信度压到 ≤0.4 |
| 任何情况 | 置信度上限 0.85（防 LLM 过度自信） |
| 任何情况 | 强制注入免责声明「AI 生成…不构成投资建议」 |
| 任何情况 | 透传 `source / fetched_at / data_complete` 溯源 |

> 护栏是**确定性逻辑**，与 DSA 的 `daily_market_context_guardrail` 思路一致——不靠 LLM 自我约束，而用代码兜底。

## 四、推送层（去 Codex）

```
本地调度(scheduler/手动) → stock_mail_plugin.send_enhanced
   → AStockSpider 采集(阶段一:真实源+降级+熔断)
   → analyzer.analyze (DeepSeek, 套护栏)        ← 原 Codex 的位置，现由本项目直接调
   → 邮件: generate_llm_report HTML 完整版 (163 SMTP)
   → 微信: format_pushplus_markdown 文字版 (PushPlus, template=markdown)
   → 飞书: 卡片(含 AI 研判摘要)
   → 快照落库(含 LLM 结论)
```
Codex 不再出现在链路中。

**PushPlus 形态**：微信走 `markdown` 文字报告（决策徽章 + 研判分 + 置信度 + 一句话 + 风险 + 催化 + 免责），清晰可读、不依赖图片——彻底避开原担心的「图片看不清」问题。

## 五、配置（config.py）

```python
DEEPSEEK_API_KEY = "YOUR_DEEPSEEK_API_KEY"
DEEPSEEK_MODEL = "deepseek-chat"
DEEPSEEK_VERIFY_MODEL = ""          # 可选 "deepseek-reasoner"
PUSHPLUS_TEMPLATE = "markdown"      # 微信侧文字报告
```
